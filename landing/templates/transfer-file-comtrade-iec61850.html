Workflow fault record

Dari event relay menuju record set yang dapat diunduh dan diperiksa.

Setiap vendor dapat memakai struktur directory dan pola nama yang berbeda. ARSAS membuat proses discovery terlihat dan tidak menganggap semua relay menyimpan file di lokasi yang sama.

01

Hubungkan relay yang dituju

Gunakan association MMS per IED agar operasi file tetap terkait dengan endpoint dan project context yang benar.

02

Probe dan browse path yang tersedia

Periksa directory dengan boundary eksplisit, cancellation dan service failure yang tetap terlihat.

03

Identifikasi disturbance record yang relevan

Tinjau filename, timestamp dan extension pasangan seperti CFG, DAT, HDR dan INF tanpa menyembunyikan record set yang tidak lengkap.

04

Unduh dan simpan evidence transfer

Pertahankan source path, destination, progress, retry boundary dan detail negative result untuk review berikutnya.

Workflow COMTRADE-aware

Perlakukan fault record sebagai satu set, bukan satu file saja.

Disturbance record yang berguna dapat terdiri dari configuration, sampled data, header dan information companion.

  • CFG untuk konfigurasi channel
  • DAT untuk sampled data
  • HDR bila relay menyediakan header deskriptif
  • INF atau companion information yang disetujui
Failure-aware engineering

Negative evidence merupakan bagian dari hasil.

Directory kosong, access rejection, timeout, unsupported service atau partial record set tidak boleh ditampilkan sebagai sukses.

  • Unsupported service tetap eksplisit
  • Timeout dan cancellation dapat dibedakan
  • Partial download tidak diterima diam-diam
  • Layout vendor-specific tidak dianggap universal

Batas engineering

File yang berhasil diunduh tetap harus divalidasi.

Transfer sukses belum membuktikan record lengkap, scaling benar atau layak untuk analisis proteksi. Periksa time basis, channel definition, record set dan konfigurasi relay sesuai prosedur.
