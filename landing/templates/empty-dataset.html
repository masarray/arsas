FCDA · coverage · dynamic recovery

An RCB can exist while its DataSet provides no useful report coverage.

Confirm the live member list rather than assuming the DataSet name represents the selected signals.

Immediate answer

Find an existing DataSet with exact coverage first.

If none exists, determine whether the IED and approved procedure permit temporary DataSet or RCB preparation. Otherwise keep the affected values in visible bounded polling.

Typical evidence
  • RCB DataSet reference is blank or unresolved
  • DataSet exists but member count is zero
  • members do not include the selected functional constraint
  • SCL file lists members not exposed by the live IED
  • write request for temporary coverage is rejected
Diagnostic path

Validate the member reference end to end.

A name match is insufficient. The report path depends on the exact object reference, functional constraint, DataSet association and RCB configuration visible on the running device.

01

Read the selected value directly

Confirm the data attribute and functional constraint exist in the live model.

02

Enumerate DataSet members

Resolve the exact live references and compare them with the selected signals.

03

Map candidate RCBs

Check which BRCBs or URCBs reference the useful DataSet and whether they are available.

04

Use explicit fallback

When no approved report coverage is available, show polling as polling rather than presenting it as report success.
