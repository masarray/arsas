MMS file service · directories · record sets

COMTRADE retrieval starts by proving the relay's file-service path.

The relay may expose disturbance records through vendor-specific directories and naming patterns. Successful transfer of one file does not prove the complete record set is present.

Immediate answer

Browse first, then download a bounded record set.

Confirm that the MMS association supports file services, enumerate the permitted directory, identify related CFG, DAT, HDR or INF files, and retain the original names and originating IED context.

Typical symptoms
  • file service is unsupported or rejected
  • root directory opens but expected records are absent
  • download starts then times out or closes
  • CFG is present but DAT is missing
  • files are transferred but belong to a different event
Diagnostic path

Treat directory and transfer results as separate evidence.

File-service availability, directory content, file size, transfer completion and record-set consistency are different checks. Preserve each result rather than reducing everything to “download failed.”

01

Confirm the selected IED session

Use the correct endpoint and association so files cannot be attributed to the wrong relay.

02

Probe file-service support

Record unsupported, access-denied and directory-not-found outcomes separately.

03

Browse within a bounded path

Retain exact names, timestamps and available size information before selecting files.

04

Validate the record set

Group matching base names and verify that the expected configuration and data components were downloaded completely.

Timeout is not the same as unsupported service.

A timeout can indicate network interruption, server behavior, large transfer duration or session closure. Keep the last confirmed offset or progress evidence when available.

Downloaded is not the same as usable.

Open the files with an approved COMTRADE viewer and verify channel definitions, sample rates, timestamps and event identity before using the record as project evidence.
