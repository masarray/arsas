Suggested workflow

Investigate the live path before changing configuration.

Commissioning pressure can encourage premature changes. Start by preserving the evidence already available from the IED and network.

01

Establish device context

Confirm the approved endpoint, active interface, device identity and live MMS model before interpreting results.

02

Observe values and events

Review selected values, quality, process timestamps, reason-for-inclusion and semantic event history.

03

Inspect reporting and GOOSE

Check RCB state, DataSet coverage, stream identity, sequence counters, retransmission and payload evidence.

04

Execute only approved operations

Use typed Direct or SBO workflows only when the procedure, role and plant condition permit active control.

05

Retain negative evidence

Unsupported services, rejected writes, report gaps, timeouts and command failures remain visible for root-cause analysis.

Boundary

Commissioning authority remains external.

ARSAS does not grant switching permission, determine plant safety, replace protection testing, approve settings or override site procedures, cybersecurity controls and independent verification.
