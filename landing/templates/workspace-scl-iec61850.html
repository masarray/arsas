Workflow configured-to-live

Jangan memperlakukan file SCL yang tersedia sebagai kebenaran tanpa pemeriksaan.

Engineering file dapat tidak lengkap, outdated atau dibuat untuk tahap integrasi yang berbeda. ARSAS memakainya sebagai configured context sambil mempertahankan apa yang benar-benar ditawarkan relay.

01

Buka sumber SCL yang tersedia

Load file proyek atau device yang disetujui dan ambil endpoint, reference, communication setting, DataSet dan reporting context.

02

Hubungkan IED yang dituju

Discovery live server model dan biarkan perbedaan tetap terlihat, bukan mengganti salah satu sumber secara diam-diam.

03

Review ketersediaan reporting

Periksa RCB existing, ownership dan verified DataSet member sebelum memilih path untuk integrasi SAS atau gateway.

04

Export baseline terkontrol

Pilih RCB dan target edition, pertahankan member yang sudah diverifikasi, lalu buat baseline tanpa blind XML deletion.

Evidence integrasi

Export SCL selected-RCB untuk legacy SAS dan gateway.

Pilih reporting path secara sengaja dan pertahankan evidence relay yang tersedia saat export.

Tujuan export

Kurangi cleanup integrasi yang sebenarnya dapat dihindari.

  • Fokus pada reporting path yang dipilih
  • Pertahankan verified DataSet member
  • Buat target edition eksplisit
  • Hindari menghapus section XML tanpa memahami relationship
  • Jaga source dan live evidence tetap traceable
Yang tidak dibuktikan

Export bukan sertifikasi interoperabilitas end-to-end.

SAS atau gateway penerima tetap harus diuji terhadap schema, edition behavior, naming, DataSet handling dan konfigurasi report yang didukung. Satu file export tidak dijamin bekerja tanpa perubahan pada semua vendor.

Berguna untuk

Integrasi multi-vendor dan troubleshooting

  • Bandingkan CID atau ICD dengan live relay model
  • Investigasi member report yang hilang atau mismatch
  • Review RCB availability sebelum gateway mapping
  • Buat baseline Edition 1 untuk legacy integration
  • Pertahankan configured dan observed context saat FAT atau SAT
Prinsip engineering

File dan perangkat adalah sumber evidence yang saling melengkapi.

Gunakan SCL untuk memahami intended configuration dan IED live untuk memahami current behavior. Perbedaan adalah evidence yang memerlukan review, bukan otomatis fault pada relay atau file.
