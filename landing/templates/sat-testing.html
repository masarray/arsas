Suggested workflow

Reconfirm what changed between factory and site.

Installation, network addressing, gateway mappings, switch configuration and final project files can introduce differences that were not present during FAT.

01

Verify endpoint access

Confirm approved IP addresses, interfaces and independent MMS associations for the installed devices.

02

Compare configured and live context

Use available SCL as project intent while checking the data model and services actually exposed by the IED.

03

Validate reports and values

Confirm required DataSet coverage, RCB state, quality, timestamps and reason-for-inclusion evidence.

04

Inspect GOOSE behavior

Check APPID, VLAN, destination MAC, sequence counters, retransmission and ordered payload against the site design.

05

Isolate integration failures

Use per-IED diagnostics to distinguish association, report, frame, gateway, file-service and command-completion problems.

Boundary

SAT remains a controlled project activity.

ARSAS does not replace approved site procedures, network-security authorization, injection testing, wiring checks, protection-setting verification, switching control or customer acceptance records.
